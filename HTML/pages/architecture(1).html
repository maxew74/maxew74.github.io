Un site web est constitué d'un ensemble de fichiers de différents types hébergés sur un serveur

  • des fichiers HTML (hypertext markup language) pour définir la structure du contenu.
  • des feuilles de styles CSS (cascading style sheets) pour mettre en forme le contenu.
  • des médias (exemples : images, vidéos, sons...)
  • des données (base de données)
  • des scripts

Le poste client reçoit le code HTML et CSS et l'interprète grâce au navigateur (IE, Firefox, Chrome, Safari, Opéra, ...)

FTP (File Transfert Protocol) : sert à uploader les fichiers du site vers le serveur.

HTTP (HyperText Transfer Protocol) : permet d'accéder au contenu HTML d'une adresse, et d'afficher le site de façon structurée sur le poste client.

Sites statiques vs dynamiques

Sites statiques : le contenu ne se met pas à jour automatiquement. Le développeur du site doit modifier la source pour modifier le contenu.

Sites dynamiques : le contenu se met à jour automatiquement ou grâce à l'utilisation d'un "back-office".

Responsive design

Un site dit "responsive" est un site dont le design est capable de s'adapter à de multiples supports (écrans d'ordinateurs, tablettes tactiles, smart-phones). Cela permet d'optimiser l'expérience utilisateur en fonction des différents formats d'écrans en conservant un unique code source.

Deux approches complémentaires :

  • Inteface flexible : les élements se redimensionnent automatiquement (utilisation d'unités relatives : %, vh, vw...)
  • Inteface responsive : plusieurs configurations sont définies (utilisation des media-querries pour définir les breakpoints)
